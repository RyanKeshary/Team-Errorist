## ğŸ¯ Overview

During disasters, **connectivity fails before people do**. CrisisMitra is designed around this reality.

The platform prioritizes:

* **Offline reliability**  
* **Fast SOS signaling**  
* **Clear situational awareness**  
* **Language accessibility across India**  

CrisisMitra runs primarily as a **static frontend application**, ensuring it works even when servers or networks are unavailable, with optional backend services planned for scaling.

## ğŸ“¶ Offline-First Architecture

**What works offline:**

* SOS alert creation & storage  
* Emergency protocols  
* Go-bag checklist  
* Emergency contacts  
* Language preference  

**How it works:**

* Critical data stored in `localStorage`  
* Connectivity listeners detect online/offline state  
* Queued SOS alerts auto-send when online  

> ğŸš§ Service Worker (true PWA support) is **planned**, not yet implemented.
